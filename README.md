# Vibe Check: Static Defect Prediction for AI-Generated Code

Can we predict whether AI-generated code will pass its test suite without running it? We train classical ML models on 123,000 labeled code samples across 57 LLMs using BigCodeBench, extracting static features from the code and comparing them against simple baselines.

The best model (Logistic Regression with TF-IDF features) achieves 0.635 AUC-ROC on a held-out test set, demonstrating that static code properties carry signal about correctness, though the ceiling is modest with hand-crafted features alone.

## Data

We use BigCodeBench (Zhuo et al., 2024), which pairs 1,140 Python programming tasks with code generated by 57 LLMs. Each sample is labeled pass or fail based on execution against a test suite. The final dataset has 123,416 rows with a 41% pass rate.

The dataset is hosted on HuggingFace at `Vihaan8/bigcodebench-sdp` (private, requires collaborator access).

```python
from huggingface_hub import hf_hub_download
import pandas as pd

path = hf_hub_download(repo_id="Vihaan8/bigcodebench-sdp", filename="processed/samples.csv", repo_type="dataset")
df = pd.read_csv(path)
```


## Pipeline

The project has three stages. Each stage produces CSV outputs that the next stage consumes.

### 1. Data Collection and Preprocessing

`data/preprocessing/collect_data.py` downloads three things from BigCodeBench: task descriptions (1,140 tasks), LLM-generated code samples (158 MB zip from GitHub releases), and per-model pass/fail labels (137 JSON files from HuggingFace). It matches samples to labels using fuzzy name normalization plus a manual mapping for edge cases, then merges everything into a single `samples.csv`.

`data/preprocessing/split_data.py` splits the dataset into train (70%), validation (15%), and test (15%) by task_id, so the same programming problem never appears in both train and test.

### 2. Feature Engineering

`feature_engineering/feature_extraction.py` extracts 16 static features from each code sample, organized into four groups:

**Classical software metrics** (3 features): lines of code, cyclomatic complexity, and max nesting depth. Computed using radon and Python's ast module.

**AST structural features** (8 features): counts of key node types (if, for, while, try, except, return, import statements) plus a binary flag for whether error handling exists.

**Prompt-code alignment features** (3 features): fraction of prompt-mentioned libraries actually imported, count of missing libraries, and the ratio of code length to prompt length.

**LLM smell features** (2 features): count of functions whose entire body is a hardcoded return, and a flag for suspiciously short solutions (5 lines or fewer).

There is also a meta feature (`meta_parse_error`) that flags code with syntax errors.

`feature_engineering/run_feature_extraction.py` runs the extraction across a full CSV and saves the output.

### 3. Model Training

Two model versions are trained, both documented in detail in `models/README.md`.

**Version 1** (`models/train_models_v1.py`): Logistic Regression and LightGBM trained on the 17 static features alone.

**Version 2** (`models/train_models_v2.py`): Logistic Regression, LightGBM, and Random Forest trained on the 17 static features plus 20,000 TF-IDF features extracted from the raw code text (10K word n-grams + 10K character n-grams).

Both versions tune hyperparameters on the validation set and evaluate on the test set. The test set is never used during training or tuning.


## How to Run

Install dependencies:

```bash
pip install pandas numpy radon scikit-learn lightgbm shap matplotlib scipy datasets requests tqdm
```

The `main.py` script orchestrates the full pipeline:

```bash
# Run everything from scratch (download data, extract features, train models)
python main.py --all

# Train all models (default, assumes data and features already exist)
python main.py

# Run individual stages
python main.py --preprocess         # download data and split
python main.py --features           # extract features from splits
python main.py --models v1          # train only v1 models
python main.py --models v2          # train only v2 models
```

You can also run each script directly:

```bash
python data/preprocessing/collect_data.py
python data/preprocessing/split_data.py --input data/clean/samples.csv --outdir data/clean/splits
python feature_engineering/run_feature_extraction.py --input data/clean/splits/train.csv --out data/clean/splits/train_features.csv
python models/train_models_v1.py
python models/train_models_v2.py
```


## Results

| Model | Features | AUC-ROC | F1 |
|---|---|---|---|
| Logistic Regression | Static (17) | 0.604 | 0.538 |
| LightGBM | Static (17) | 0.608 | 0.516 |
| Logistic Regression | Static + TF-IDF (20,017) | 0.635 | 0.537 |
| LightGBM | Static + TF-IDF (20,017) | 0.634 | 0.529 |
| Random Forest | Static (17) | 0.605 | 0.537 |

Adding TF-IDF improved AUC by about 0.03 across models by giving them access to actual code tokens rather than just summary statistics. The most predictive static features are lines of code, presence of error handling, and cyclomatic complexity. All correlate negatively with the pass label, meaning more complex solutions are more likely to fail.
